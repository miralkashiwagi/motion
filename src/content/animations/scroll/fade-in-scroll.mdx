# スクロールフェードイン効果

スクロールしてページ内の要素が画面に入ったときに、徐々にフェードインするエフェクトを実装します。

## HTML構造

```html
<div class="container">
  <section class="fade-section">
    <h2>セクション 1</h2>
    <p>スクロールするとフェードインします</p>
  </section>
  
  <section class="fade-section">
    <h2>セクション 2</h2>
    <p>このセクションもフェードインします</p>
  </section>
  
  <section class="fade-section">
    <h2>セクション 3</h2>
    <p>最後のセクションです</p>
  </section>
</div>
```

## CSS実装

```css
.container {
  max-width: 800px;
  margin: 0 auto;
  padding: 20px;
}

.fade-section {
  height: 300px;
  margin: 100px 0;
  padding: 30px;
  background-color: #f3f4f6;
  border-radius: 8px;
  opacity: 0;
  transform: translateY(20px);
  transition: opacity 0.6s ease, transform 0.6s ease;
}

.fade-section.visible {
  opacity: 1;
  transform: translateY(0);
}

h2 {
  margin-bottom: 15px;
  color: #1f2937;
}

p {
  color: #4b5563;
}
```

## JavaScript実装

```javascript
document.addEventListener('DOMContentLoaded', function() {
  const fadeSections = document.querySelectorAll('.fade-section');
  
  // Intersection Observerの設定
  const observer = new IntersectionObserver((entries) => {
    entries.forEach(entry => {
      // 要素が画面内に入ったらvisibleクラスを追加
      if (entry.isIntersecting) {
        entry.target.classList.add('visible');
        // 一度表示したら監視を解除（オプション）
        observer.unobserve(entry.target);
      }
    });
  }, {
    // 要素が20%以上表示されたらコールバックを実行
    threshold: 0.2,
    // ビューポートを基準にする
    rootMargin: '0px'
  });
  
  // 各要素を監視対象に追加
  fadeSections.forEach(section => {
    observer.observe(section);
  });
});
```

## 解説

このアニメーションは、モダンなウェブサイトでよく見られるスクロールトリガーエフェクトの一種です。

1. 初期状態では要素は透明度0で下に少しずれた位置にあります
2. Intersection Observerを使って要素が画面内に入ったかを検知します
3. 要素が画面内に入ると、CSSトランジションによって徐々に表示され、元の位置に移動します

Intersection Observerを使うことで、スクロールイベントを直接監視するよりもパフォーマンスが向上します。
